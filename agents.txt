WHAT THIS PROJECT IS
--------------------
Signalor is an AI GEO (Generative Engine Optimization) platform. It tracks
how brands appear in AI search engines (ChatGPT, Claude, Gemini, Perplexity,
Google AI Overviews, Bing AI), analyzes citation quality, scores content
against SEO/GEO pillars, and gives actionable recommendations to improve AI
visibility.

Core features:
- AI visibility tracking across 6 engines
- Multi-pillar GEO score (technical, content, citations, sentiment, competitors)
- URL/content analysis with async run-based job system (polled)
- Competitor benchmarking
- Backlink marketplace (static curated provider directory — no purchase API)
- Content optimization recommendations
- Gamification (points, badges, leaderboard)
- Creator/affiliate referral program
- Blog (Sanity CMS)
- WordPress & Shopify integrations
- Payments via Dodo Payments
- Error monitoring via Sentry (EU ingest)

CODING CONVENTIONS
------------------

Package manager:
  Always pnpm. Never npm or yarn.

Icons:
  Import from @/components/icons, NOT lucide-react.
  CORRECT: import { AlertCircle, Loader2 } from "@/components/icons";
  WRONG:   import { AlertCircle } from "lucide-react";

API calls:
  All backend calls go through apiClient or apiClientLong from
  src/lib/api/client.ts. Never use raw fetch or axios in components.
  Always validate responses with Zod schemas.

Data fetching:
  New components MUST use TanStack Query, not useState + useEffect.
  Canonical pattern: see src/components/dashboard/ai-recommendation-card.tsx

  const { data, isLoading, error } = useQuery({
    queryKey: ["things", id],
    queryFn: () => getThings(id),
  });

Client vs Server components:
  Default to Server Components.
  Add "use client" only when using: useState, useEffect, useQuery,
  browser APIs, or event handlers.

Comments:
  Write no comments by default. Only add one when the WHY is non-obvious
  (hidden constraint, workaround for a bug). Never describe what the code does.

TypeScript:
  Strict mode on. No "any" types. Validate all API responses with Zod.

Styling:
  Tailwind utility classes only — no inline styles, no CSS modules.
  Theme is managed by next-themes. Root html starts as className="light".
  Use cn() from @/lib/utils for conditional class merging.

Env vars:
  Client-side must be prefixed NEXT_PUBLIC_.
  Source of truth: env/ folder. Switch with pnpm env:<target>.
  All validated via Zod in src/lib/env.ts.

SENTRY ERROR MONITORING
-----------------------

              NOT the old sentry.client.config.ts pattern

Client init : src/instrumentation-client.ts
              - replayIntegration() for Session Replay
              - tunnel: "/api/tunnel" (valid — browsers use relative URLs)
              - sendDefaultPii: true
              - tracesSampleRate: 1 (always 1 on client)
              - replaysSessionSampleRate: 0.1
              - replaysOnErrorSampleRate: 1.0

Server init : sentry.server.config.ts (via instrumentation.ts, nodejs runtime)
              - NO tunnel option (servers connect directly to Sentry ingest)
              - tracesSampleRate: env-conditional (1.0 dev, 0.1 prod)
              - includeLocalVariables: true

Edge init   : sentry.edge.config.ts (via instrumentation.ts, edge runtime)
              - NO tunnel option
              - tracesSampleRate: env-conditional

Tunnel      : src/app/api/tunnel/route.ts (POST)
              Proxies browser Sentry envelopes through Next.js to bypass
              ad-blockers. Browsers point to /api/tunnel.

COMMON PITFALLS
---------------
1. Sanity Studio MUST use next/dynamic({ ssr: false })
   It references window at import time — SSR will crash.

2. Date.toLocaleString() without a fixed locale causes SSR hydration mismatch.
   Always pass "en-US" locale with UTC options.

3. Vercel "Sensitive" env vars are NOT inlined into NEXT_PUBLIC_* builds.
   Uncheck "Sensitive" for any NEXT_PUBLIC_ variable in Vercel dashboard.

4. pg and better-sqlite3 are serverExternalPackages in next.config.ts.
   Never import them in any client component.

5. Dev uses Turbopack (pnpm dev:turbo). Production always uses webpack
   (pnpm build). Some Turbopack quirks do not appear in production builds.

6. After deleting a route, Next.js may cache stale TypeScript types.
   Delete .next/dev/types/app/<route>/ manually to clear the cache.

7. Amplitude CSP: sr-client-cfg.amplitude.com and gs.amplitude.com must be
   in the PRODUCTION connect-src, not inside the dev-only devConnectSrc block.

8. Sentry EU DSN: *.sentry.io only matches ONE subdomain level.
   The EU ingest host has 3 levels: o4511432219688960.ingest.de.sentry.io
   Always add *.de.sentry.io separately in CSP connect-src.

9. Sentry tunnel option: valid ONLY in instrumentation-client.ts (browser).
   sentry.server.config.ts and sentry.edge.config.ts must NOT have tunnel.
   Server-side Sentry connects directly to ingest.

10. No middleware.ts exists in this project.
    Route protection is handled client-side inside page components.
